# Установка Claude Code + VS Code + GitHub: полная инструкция

Дата актуальности: май 2026. Источники: официальная документация Claude Code
(code.claude.com/docs) и github.com/anthropics/claude-code.

---

## Что понадобится (один раз)

- **Аккаунт Claude с подпиской Pro/Max** (или Team/Enterprise/Console).
  ⚠️ Бесплатный план Claude.ai НЕ включает Claude Code.
- **VS Code** (свежая версия) — https://code.visualstudio.com
- **Git** — https://git-scm.com/downloads
  (на macOS часто уже стоит; проверь `git --version`).
- **Аккаунт GitHub** — https://github.com
- Node.js **НЕ нужен** — нативный установщик Claude Code не требует Node
  (это изменилось; npm-метод теперь устаревший).

---

## ШАГ 1. Установить Claude Code (нативный установщик — рекомендуется)

Открой терминал и выполни команду под свою ОС.

**macOS / Linux:**
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows (PowerShell, не CMD):**
```powershell
irm https://claude.ai/install.ps1 | iex
```
> Подсказка: если видишь `PS C:\>` — это PowerShell (правильно).
> Если просто `C:\>` — это CMD, открой PowerShell.
> На Windows желательно поставить ещё «Git for Windows», чтобы Claude Code мог
> пользоваться bash-инструментами.

**Проверка установки** (закрой и открой терминал заново, потом):
```bash
claude --version
```
Если `command not found` — полностью закрой окно терминала и открой новое.
Для диагностики проблем: `claude doctor`.

**Первый запуск и вход:**
```bash
claude
```
Откроется браузер → выбери **«Claude account with subscription»** (НЕ Anthropic Console,
если у тебя обычная подписка) → войди → готово, токен подхватится автоматически.

---

## ШАГ 2. Поставить расширение «Claude Code for VS Code»

1. Открой VS Code.
2. Слева на панели нажми **Extensions** (значок кубиков) или Ctrl/Cmd+Shift+X.
3. В поиске набери **Claude Code** и установи официальное расширение
   от **Anthropic** (издатель — Anthropic).
4. Перезапусти VS Code.
5. Появится боковая панель Claude Code (значок в левой панели). Открой её —
   расширение само подхватит уже установленный CLI и твою авторизацию.

> Если панель пишет «No available IDEs detected» или просит Node — обычно дело в PATH.
> Закрой VS Code полностью, открой терминал, убедись что `claude --version` работает,
> и запусти VS Code заново. На крайний случай можно запускать `claude` прямо во
> встроенном терминале VS Code (Terminal → New Terminal).

---

## ШАГ 3. Создать папку проекта и открыть её в VS Code

В терминале (или через VS Code → File → Open Folder):
```bash
# выбери удобное место, например Рабочий стол
cd ~/Desktop
mkdir kt-ads-quiz
cd kt-ads-quiz
```
Затем в VS Code: **File → Open Folder…** → выбери папку `kt-ads-quiz`.

---

## ШАГ 4. Положить файлы данных в проект (ВАЖНО — до запуска промпта)

Скопируй в папку `kt-ads-quiz` два файла, которые я тебе дал:
- `ads_question_bank.json`
- `topics.json`

Можно прямо в корень — Claude Code сам перенесёт их в `data/` по инструкции из промпта.
(Промпт ссылается на эти файлы, поэтому они должны лежать в проекте заранее.)

---

## ШАГ 5. Запустить Claude Code и отдать ему промпт

1. В боковой панели Claude Code (или во встроенном терминале командой `claude`)
   начни новую сессию — убедись, что рабочая папка = `kt-ads-quiz`.
2. Открой файл `PROMPT_for_Claude_Code.md`, скопируй текст промпта
   (от «=== НАЧАЛО ПРОМПТА ===» до «=== КОНЕЦ ПРОМПТА ===»)
   и вставь в чат Claude Code. Отправь.
3. Claude Code начнёт создавать файлы. Он будет показывать изменения (diff) и
   спрашивать разрешение на запись/команды — подтверждай.
4. Когда закончит — проверь локально (см. Шаг 6).

> Совет: после первого ответа можешь попросить «покажи дерево файлов» и
> «запусти локальный сервер, я проверю». Если что-то не так — пиши Claude Code
> прямо что поправить, он отредактирует файлы.

---

## ШАГ 6. Проверить сайт локально

Во встроенном терминале VS Code (Terminal → New Terminal), в папке проекта:
```bash
# если есть python3:
python3 -m http.server 8000
```
Открой в браузере http://localhost:8000 — потыкай все три кнопки, тест, темы.
Останови сервер: Ctrl+C.

> Почему сервер, а не двойной клик по index.html: `fetch` к JSON не работает с
> протокола file://, нужен http. На GitHub Pages это уже http, там всё ок.

## ШАГ 7. Создать репозиторий на GitHub и запушить

### Вариант А (проще) — через сайт GitHub + командную строку

1. Зайди на https://github.com → справа вверху **+** → **New repository**.
2. Имя: например `kt-ads-quiz`. Сделай **Public** (для бесплатного GitHub Pages).
   НЕ добавляй README/gitignore (они уже будут в проекте). Нажми **Create repository**.
3. GitHub покажет адрес репозитория. Скопируй HTTPS-адрес вида
   `https://github.com/ТВОЙ_ЛОГИН/kt-ads-quiz.git`.
4. В терминале VS Code (в папке проекта) выполни по порядку:
```bash
git init
git add .
git commit -m "Первая версия: КТ тренажёр по АиСД"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/kt-ads-quiz.git
git push -u origin main
```
При первом push GitHub попросит авторизацию — войди через браузер (или Personal
Access Token). VS Code обычно сам предложит залогиниться через GitHub.

> Можно проще: попроси Claude Code — «выполни git init, добавь всё, сделай коммит
> и подскажи команды для push» — он умеет работать с git и предложит/выполнит команды.

### Вариант Б — через интерфейс VS Code

1. Слева на панели **Source Control** (значок ветки) или Ctrl/Cmd+Shift+G.
2. Нажми **Initialize Repository**.
3. Внизу впиши сообщение коммита → галочка **Commit**.
4. Нажми **Publish Branch** → VS Code предложит создать репозиторий на GitHub
   (выбери Public) и сам всё запушит.

---

## ШАГ 8. Включить GitHub Pages (чтобы был публичный адрес)

1. На GitHub открой свой репозиторий → **Settings** → слева **Pages**.
2. В «Build and deployment»: Source = **Deploy from a branch**.
3. Branch = **main**, папка = **/ (root)** → **Save**.
4. Подожди 1–2 минуты. Адрес сайта появится там же:
   `https://ТВОЙ_ЛОГИН.github.io/kt-ads-quiz/`
5. Открой этот адрес на iPhone в Safari.

> Поэтому в промпте требование «относительные пути»: на Pages сайт живёт в подпапке
> `/kt-ads-quiz/`, и пути от корня `/` сломались бы.

---

## ШАГ 9. Установить как приложение на iPhone (PWA)

1. Открой адрес `https://ТВОЙ_ЛОГИН.github.io/kt-ads-quiz/` в **Safari** (именно Safari).
2. Нажми кнопку **«Поделиться»** (квадрат со стрелкой вверх).
3. Выбери **«На экран „Домой"»** (Add to Home Screen).
4. На рабочем столе появится иконка — запускается в полноэкранном режиме, как приложение,
   и работает офлайн после первого открытия.

---

## Как обновлять сайт потом (новый цикл)

Когда добавишь новые вопросы или Claude Code что-то поправит:
```bash
git add .
git commit -m "что изменил"
git push
```
GitHub Pages обновится автоматически через минуту. На iPhone обнови страницу в приложении
(service worker подтянет новую версию; иногда нужно закрыть и открыть приложение).

## Частые проблемы

- **`claude: command not found`** → закрой терминал полностью, открой новый;
  если не помогло — `claude doctor`.
- **VS Code-панель просит Node** → запускай `claude` во встроенном терминале;
  убедись, что CLI установлен нативным способом и виден в PATH.
- **JSON не грузится при двойном клике по index.html** → нужен локальный http-сервер
  (Шаг 6) либо GitHub Pages; с file:// fetch заблокирован браузером.
- **На GitHub Pages пустая страница / 404 на ресурсы** → значит где-то остались
  пути от корня `/`; должны быть относительные (`./`, `css/...`, `data/...`).
- **Push отклонён (auth)** → залогинься в GitHub через VS Code (он предложит окно входа)
  или создай Personal Access Token в GitHub → Settings → Developer settings.
