# КТ • Алгоритмы и структуры данных

PWA-тренажёр для подготовки к комплексному тестированию (КТ) в магистратуру Республики
Казахстан, группа образовательных программ **М094 «Информационные технологии»**, дисциплина
**«Алгоритмы и структуры данных»**.

Чистая статика: **HTML + CSS + ванильный JavaScript (ES-модули)**. Без фреймворков, сборщиков,
npm-зависимостей и бэкенда. Работает офлайн (Service Worker), ставится на домашний экран iPhone.

## Возможности

- **168 выверенных вопросов** с подробным разбором (`explanation`) к каждому и блоками кода.
- **10 тем** с конспектом (markdown), в т.ч. дописанные темы 6 (сложность) и 8 (хеш-таблицы).
- **4 режима теста:** блиц (15), полный тест формата КТ (30, с таймером), квиз по теме, тест по блоку.
- **Стратифицированная рандомизация** — вопросы равномерно распределены по темам.
- **Прогресс** в `localStorage`: чтение конспекта (до 30 баллов) + аттестация квизом (до 70).
- Тёмная «техно»-тема, адаптив, тач-таргеты ≥ 44px.
- Самопроверка готовности: открой **`tests.html`**.

## Структура

```
index.html          Главное меню (3 кнопки)
topics.html         Список тем с прогресс-барами
topic.html?id=N     Тема: блоки конспекта + тесты блоков + квиз темы
quiz.html           Движок теста: ?mode=blitz|full|topic|block&count=N&topic=N&block=ID
tests.html          Автотесты готовности (открыть в браузере)
css/styles.css      Дизайн-система
js/
  data-loader.js    Загрузка JSON + кеш
  progress.js       Прогресс в localStorage
  quiz-engine.js    Стратификация (Fisher–Yates + round-robin), скоринг, таймер
  topics.js         Рендер списка тем и страницы темы (markdown)
  quiz.js           Контроллер страницы теста
  app.js            Утилиты, markdown-рендер, регистрация Service Worker
data/
  ads_question_bank.json   Банк вопросов (с explanation/code)
  topics.json              Конспект по темам
icons/              icon-192.png, icon-512.png, apple-touch-icon.png
manifest.json       PWA-манифест
service-worker.js   Офлайн-кеш (cache-first статика, network-first данные)
```

## Локальный запуск

Из-за того что страницы грузят JSON через `fetch`, открывать `index.html` двойным кликом
(`file://`) нельзя — нужен локальный HTTP-сервер:

```bash
# Python 3
python -m http.server 8000
# затем открыть http://localhost:8000/
```

Проверить готовность: открыть `http://localhost:8000/tests.html` — все проверки должны быть зелёными.

## Установка на iPhone (PWA)

1. Открыть сайт в **Safari**.
2. Кнопка «Поделиться» → **«На экран Домой»**.
3. Приложение запустится в полноэкранном режиме (standalone) и будет работать офлайн
   после первого визита.

## Публикация на GitHub Pages

Все пути **относительные** (`./`, `css/...`, `data/...`), поэтому проект работает и в
подпапке `https://<user>.github.io/<repo>/` без шага сборки.

```bash
git init
git add .
git commit -m "КТ АиСД: PWA-тренажёр"
git branch -M main
git remote add origin https://github.com/<user>/<repo>.git
git push -u origin main
```

Затем в настройках репозитория: **Settings → Pages → Build and deployment → Source: Deploy from a
branch → Branch: `main` / `(root)`**. Через минуту сайт будет доступен по адресу
`https://<user>.github.io/<repo>/`.

## Формула прогресса темы (0–100)

- до **30** баллов: `(прочитано блоков / всего блоков) × 30`;
- до **70** баллов: `(лучший результат квиза темы) × 70`.

100/100 достигается только если прочитаны все блоки **и** квиз темы пройден на 100%.
Сброс прогресса — кнопкой в футере главного экрана.
