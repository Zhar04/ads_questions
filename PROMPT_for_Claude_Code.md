=== НАЧАЛО ПРОМПТА ===

Ты — старший фронтенд-разработчик. Построй PWA-приложение (Progressive Web App) для подготовки
к экзамену КТ (комплексное тестирование) в магистратуру Республики Казахстан, группа
образовательных программ **М094 «Информационные технологии»**, дисциплина
**«Алгоритмы и структуры данных»**.

## КРИТИЧЕСКИЕ ТРЕБОВАНИЯ К СТЕКУ

- **Только статика: HTML + CSS + ванильный JavaScript (ES6 модули). НИКАКОГО фреймворка,
  сборщика, npm-зависимостей, бэкенда.** Сайт должен работать как статические файлы,
  открытые напрямую, и на GitHub Pages без шага сборки.
- Данные грузятся из JSON-файлов в папке `data/` через `fetch`.
- Состояние/прогресс хранится в `localStorage` (НЕ использовать никакие внешние БД).
- Язык интерфейса — **только русский**.
- Должно идеально работать на мобильном (iPhone, Safari), адаптивная вёрстка, тач-таргеты ≥ 44px.
- Это **PWA**: добавь `manifest.json`, `service-worker.js` и иконки, чтобы на iPhone
  через Safari → «Поделиться» → «На экран Домой» приложение ставилось как нативное,
  запускалось в полноэкранном режиме (standalone) и работало офлайн после первого визита.

## ИСХОДНЫЕ ДАННЫЕ (уже лежат в папке проекта)

1. `ads_question_bank.json` — банк из 109 выверенных вопросов. Схема каждого вопроса:
   ```json
   {
     "id": "ADS-001",
     "question": "текст вопроса",
     "options": [{"letter":"A","text":"..."}, ...],   // 5 вариантов A–E (иногда 4)
     "correct_answers": ["B"],        // массив; для КТ всегда ОДИН верный
     "answer_type": "single",
     "topic_id": 5,                   // 1..10, ключ к теме
     "topic": "Алгоритмы сортировки и поиска",
     "difficulty": "В",               // А / В / С
     "verification": "verified|corrected|source",
     "has_figure": false,             // true => вопрос про рисунок, которого нет в тексте
     "source_files": [...],
     "original_keys": {...}
   }
   ```
2. `topics.json` — конспект, нарезанный на темы и блоки. Схема:
   ```json
   {
     "meta": {...},
     "topics": [
       {
         "topic_id": 1,
         "title": "Базовый процедурно-ориентированный алгоритмический язык",
         "blocks": [
           {"block_id":"t1_b1","title":"Алфавит языка","content_md":"...markdown...","links":[]},
           ...
         ],
         "needs_notes": false   // если true — конспект по теме отсутствует, нужно написать
       }
     ]
   }
   ```

## ЭКРАН 1 — ГЛАВНОЕ МЕНЮ (index.html)

Три большие кнопки-карточки:
1. **Темы** → topics.html
2. **Блиц-тест** (15 вопросов) → quiz.html?mode=blitz&count=15
3. **Полный тест** (30 вопросов, как на КТ) → quiz.html?mode=full&count=30

Сверху: заголовок дисциплины, бейдж «КТ • Магистратура • М094».
Под кнопками — общий прогресс (средний % по всем темам) и краткая статистика
(пройдено тем, всего вопросов в банке, лучший результат полного теста).

## ЭКРАН 2 — ТЕМЫ (topics.html)

Список из 10 тем. Для каждой темы — карточка с:
- названием темы и номером;
- **прогресс-баром** 0–100 (значение из localStorage, формула ниже);
- меткой «X/Y блоков прочитано» и «лучший результат квиза темы: Z%».
Клик по теме → topic.html?id=N

## ЭКРАН 3 — СТРАНИЦА ТЕМЫ (topic.html?id=N)

- Сверху **закреплённая (sticky) кнопка «Пройти квиз по теме»** →
  quiz.html?mode=topic&topic=N&count=15 (дай также вариант на 30 вопросов, если их хватает в банке).
- Ниже — список **блоков** темы. Каждый блок:
  - заголовок;
  - конспект (рендери `content_md` как markdown: заголовки, списки, **жирный**,
    блоки кода ```...``` с моноширинным шрифтом и горизонтальным скроллом);
  - если есть `links` — выведи как ссылки «Доп. материалы»;
  - в конце блока — кнопка **«Тест по блоку (5–10 вопросов)»**. Это ФИКСИРОВАННЫЙ
    (не рандомный) маленький тест строго по содержанию блока.
- Отмечай блок «прочитан», когда пользователь докрутил его до конца (IntersectionObserver)
  ИЛИ нажал явную кнопку «Отметить прочитанным». Сохраняй в localStorage.
- Если у темы `needs_notes:true` (темы 6 и 8) — покажи аккуратную плашку
  «Конспект в разработке» и сгенерируй для этих тем базовый конспект сам
  (по официальной программе: тема 6 — константная/линейная/логарифмическая/
  линейно-логарифмическая/квадратичная/кубическая/экспоненциальная/факториальная
  сложности, нотация O; тема 8 — хеш-функции, коэффициент заполнения, коллизии:
  цепочки, открытая адресация, линейное/квадратичное зондирование, двойное хеширование).
  Добавь эти блоки прямо в data/topics.json.

## ЭКРАН 4 — ДВИЖОК ТЕСТА (quiz.html), 4 режима через query-параметры

Параметры: `?mode=blitz|full|topic|block` `&count=N` `&topic=N` `&block=ID`

Общая логика для всех режимов:
- Один вопрос на экране (или список — на твой вкус, но мобильно удобно), варианты A–E кнопками.
- Прогресс «Вопрос k из N» + прогресс-бар.
- Кнопка **«Показать ответ»** на КАЖДОМ вопросе — по нажатию подсвечивает верный
  вариант и показывает поле `explanation` (объяснение, почему ответ верный). По умолчанию
  ответ скрыт.
- В конце — экран результата: % верных, разбор всех вопросов (верный/неверный + объяснение),
  кнопки «Пройти заново» и «На главную».

Различия режимов:
- **blitz**: 15 случайных вопросов из ВСЕГО банка. Ответы и объяснения можно показывать
  сразу после ответа на каждый вопрос (обучающий режим).
- **full**: 30 вопросов из всего банка, формат КТ. **Таймер** (предложи ~45–60 минут,
  сделай настраиваемым в коде константой). Правильные ответы по умолчанию показываются
  ТОЛЬКО В КОНЦЕ (как на реальном КТ), но кнопка «Показать ответ» доступна на каждом
  вопросе, если пользователь захочет подсмотреть. В конце — «протокол ответов» как на КТ.
- **topic**: вопросы только с данным `topic_id`, 15 или 30, случайно.
- **block**: фиксированные вопросы конкретного блока (по 5–10, не случайные).

### ⚠️ КЛЮЧЕВОЕ: стратифицированная рандомизация (НЕ обычный random!)

Для режимов blitz и full набор вопросов ОБЯЗАН быть равномерно распределён по темам,
чтобы не получилось «半 теста из одной темы». Реализуй так:
- сгруппируй банк по topic_id;
- перемешай вопросы внутри каждой темы (Fisher–Yates);
- набирай вопросы по кругу (round-robin) по темам, пока не наберёшь count;
- если в какой-то теме вопросы кончились — продолжай добирать из остальных;
- итоговый список ещё раз перемешай по порядку показа.
Так покрытие тем максимально равномерное.

Пропускай вопросы с `has_figure:true` в авто-наборе (или помечай «вопрос с рисунком»),
т.к. рисунка нет — но в режиме topic/block их можно показывать с пометкой.

## ФОРМУЛА ПРОГРЕССА (progress.js, localStorage)

Прогресс темы = **0..100**:
- **до 30 баллов** за чтение конспекта: (прочитано блоков / всего блоков) × 30.
- **до 70 баллов** за «аттестацию темы» — лучший результат квиза темы (mode=topic):
  (доля верных ответов) × 70.
- Итого 100/100 достигается только если ВСЕ блоки прочитаны И квиз пройден на 100%.
Храни в localStorage по ключу вроде `progress:topic:{id}` структуру
`{ blocksRead: ["t1_b1",...], bestTopicQuizPct: 0.0 }`.
Также храни лучший результат полного теста: `progress:fullTestBest`.
Сделай кнопку сброса прогресса в настройках/футере.

## ОБЪЯСНЕНИЯ К ВОПРОСАМ (ОБЯЗАТЕЛЬНО ко всем)

В банке пока НЕТ поля `explanation`. Добавь в data/ads_question_bank.json к КАЖДОМУ
вопросу поле `"explanation"` — содержательный разбор НА РУССКОМ: почему верный ответ
правильный и (кратко) почему остальные неверны. НЕ пиши воду вида «ответ B, потому что
это правильно». Используй знания по алгоритмам и структурам данных (источники курса:
Лафоре, Ахо-Хопкрофт-Ульман, Кнут, Седжвик, Deitel). Для вопросов с кодом C++ —
объясни, что делает код. Если в вопросе есть фрагмент кода, продублируй его в новое
поле `"code"` для отдельного отображения моноширинным блоком.

## PWA-ДЕТАЛИ

- `manifest.json`: name «КТ • Алгоритмы и структуры данных», short_name «КТ АиСД»,
  start_url «./», display «standalone», background_color «#0a0a0f»,
  theme_color «#7c3aed», иконки 192 и 512.
- В `<head>` всех HTML добавь: `<link rel="manifest" href="manifest.json">`,
  `<meta name="theme-color" content="#7c3aed">`,
  `<link rel="apple-touch-icon" href="icons/apple-touch-icon.png">`,
  `<meta name="apple-mobile-web-app-capable" content="yes">`,
  `<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">`.
- `service-worker.js`: кешируй все HTML/CSS/JS/JSON/иконки (cache-first для статики,
  network-first для JSON-данных, с фолбэком на кеш офлайн). Регистрируй SW в app.js.
- Сгенерируй простые иконки сам (можно SVG→PNG или нарисуй на canvas скриптом):
  тёмный фон, символ дерева/графа или буквы «АиСД» акцентным цветом.
- ВАЖНО про GitHub Pages: пути должны быть ОТНОСИТЕЛЬНЫЕ (`./`, `css/...`, `data/...`),
  чтобы работало в подпапке `https://user.github.io/repo-name/`. Не используй пути от корня `/`.
